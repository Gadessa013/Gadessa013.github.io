How to Structure Posts

To add content to this gallery, you must structure your files within the /posts directory of the configured GitHub repository.

1. Simple Post

A simple post is a single Markdown file placed directly inside the /posts directory.

/posts
  └── my-first-post.md

2. Complex Post

A complex post is a folder within /posts. This folder must contain exactly one Markdown file, which will be treated as the main content. Any other files (images, PDFs, etc.) will be listed as downloadable reference files.

Relative links to images within your Markdown (e.g., ![My Image](./diagram.png)) will be automatically resolved.

/posts
  └── my-complex-project/
      ├── main-article.md
      ├── diagram.png
      └── data-sheet.pdf
